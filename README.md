# AI Insurance Claim Co-Pilot

An AI Co-Pilot that helps insurance officers evaluate vehicle damage claims,
built for the "AI Co-Pilot for Industry" hackathon theme.

## What it does

- **Image (CNN)**: detects vehicle damage from an uploaded photo.
- **PDF (OCR/regex extraction)**: pulls policy number, claim number, customer
  name, and incident details from the uploaded claim document.
- **Tabular (ANN)**: predicts claim risk level (Low / Medium / High) from
  claim amount, vehicle age, previous claims, driver age, and damage severity.
- **Explainable AI**: Grad-CAM heatmap for the CNN, permutation-based feature
  importance for the ANN, plus confidence scores for every prediction.
- **LLM Co-Pilot**: summarizes the claim and suggests a next action in plain
  language (never used as the predictive model itself).
- **Human-in-the-Loop**: the officer must approve, reject, or modify every
  AI recommendation. Nothing is auto-approved.
- **Report**: generates a downloadable PDF with all inputs, predictions,
  explanations, and the human's final decision.

## Project structure

```
insurance_copilot/
├── app.py                  # Streamlit app (all pages)
├── train_models.py         # Trains the CNN + ANN models (run once)
├── requirements.txt
├── runtime.txt             # Pins Python 3.11 for Streamlit Cloud
├── models/                 # Saved .keras models + scaler (generated)
├── data/                   # Synthetic training data + decision log (generated)
└── utils/
    ├── pdf_extract.py       # PDF text + field extraction
    ├── explainability.py    # Grad-CAM + feature importance
    ├── copilot.py            # LLM summary/recommendation
    ├── report_generator.py  # PDF report builder
    └── storage.py            # Local JSON "database" for human decisions
```

## Running locally

```bash
pip install -r requirements.txt
python train_models.py     # trains and saves the CNN + ANN models
streamlit run app.py
```


## Notes on the data

Since no real labeled dataset was available for the hackathon, both models
are trained on synthetic data generated by `train_models.py`. The code is
written so a real dataset (real claim photos + real historical claims CSV)
can be swapped in later without changing the model architecture, the app,
or any other file.

## Swapping in Firebase

`utils/storage.py` currently uses a local JSON file to store human review
decisions, to keep the demo simple and avoid credential setup. It exposes
two functions (`save_decision`, `get_all_decisions`) — swap the internals
of these two functions for Firebase calls to switch to a real database
without touching `app.py`.
